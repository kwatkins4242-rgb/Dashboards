1. Policy subagent (triggers + behavior)
.cursor/agents/jarvis-vision-policy.md (jarvis-vision-policy) encodes:

Wake: screen snapshot + webcam still when possible; how Jarvis weighs both with the rest of context.
Emergencies: respond immediately with safety-forward wording.
Other people visible: explicitly offer voice vs mute + text.
Trigger table: wake phrases, NL “what do you see”, 5‑minute rhythm, motion-after-still notes, camera failures.
Retention: rolling vs vault vs quarantine; PII flows (quarantine until you review—no echoed secrets).
Delegate it when vision, wake flows, privacy, or “what should Jarvis say here?” need a consistent playbook.

2. API support in core/senses_api.py
Screenshots now land under ODIN_SCREENSHOTS_DIR/rolling/ (not the root of that folder). Old behavior at the root is unchanged for files already there; new writes use rolling/.

TTL: purge_rolling_screenshots() deletes captures in rolling/ older than ODIN_SCREENSHOT_RETENTION_DAYS (default 7). vault/ and quarantine/ are never auto-deleted.
GET /wake/vision-bundle: grabs screen PNG + optional camera JPEG, saves both under rolling/, returns base64 for wiring to wake/word flow.
POST /screenshot/promote_to_vault?filename=…: basename only, moves rolling/ → vault/.
POST /screenshot/quarantine?filename=…: rolling/ → quarantine/ for PII holds.

Here is what was added.

1. wake_word/vision_bundle_async.py
async def fetch_wake_vision_bundle(base_url, timeout=20) — httpx.AsyncClient GET to {base}/wake/vision-bundle.
fetch_wake_vision_bundle_blocking() — runs asyncio.run(...) so normal sync code can await the full request without an existing loop (do not use this inside an already-running asyncio loop; call the async function with await there instead).
schedule_wake_vision_bundle() — starts a daemon thread that runs the blocking helper so Porcupine’s loop is not stalled. Optional callbacks: on_complete, on_error. Default logs filenames or camera errors.
Defaults for base URL: M2_SENSES_URL or JARVIS_SENSES_URL, then http://127.0.0.1:8000.

2. wake_word/wake_word_detector.py
Fixed invalid first line (# WAKE WORD DETECTOR).
New optional kwarg: vision_bundle_base_url (overrides env if set).
Reads M2_SENSES_URL / JARVIS_SENSES_URL when the kwarg is omitted; if still empty, vision bundle is not called.
After every on_wake(...) (real Porcupine path and simulation), calls schedule_wake_vision_bundle() so the asyncio fetch runs in the background.
3. wake_word/__init__.py
Makes wake_word a proper package so . imports resolve.
Configure: set M2_SENSES_URL=http://127:port (same base as your FastAPI app that mounts the senses router) so wake triggers hit /wake/vision-bundle.
